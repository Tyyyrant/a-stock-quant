# 涨停板分析框架 — 开发文档

## 1. 当前系统对涨停的态度

**现状：系统性歧视涨停票。**

```
涨停 → 偏离MA20大 → 过热惩罚扣分 → 量价异常 → 7-Agent判SELL/HOLD
```

结果：诺德股份(涨停+10%)、盛剑科技(涨停+10%)、光华科技(涨停+10%) 全部被毙。
但实际上这些票有资金在做，涨停是市场用真金白银投的票。

**核心矛盾**：系统把"涨停"等同于"风险"，但 A 股中涨停 = 资金认可 = 辨识度。

---

## 2. 涨停分析框架

不是所有涨停都值得买。需要区分**真涨停（资金吃筹）**和**假涨停（诱多出货）**。

### 2.1 涨停质量六维评分

| 维度 | 满分场景 | 扣分场景 |
|------|---------|---------|
| **封板时间** | 开盘30分钟内封板，全天不开 | 尾盘偷袭封板（14:30后） |
| **封单强度** | 封单/日均成交 > 2倍 | 封单稀少，多次炸板 |
| **涨停位置** | 低位首板（横盘后第一个涨停） | 高位加速板（已有50%+涨幅后再涨停） |
| **量价配合** | 量比1.5~3倍，温和放量涨停 | 一字无量（买不到）或巨量>5倍（出货） |
| **板块地位** | 板块内最早封板（龙头） | 跟风涨停（板块涨了3天它才涨停） |
| **K线形态** | 突破底部平台/关键位的首板 | 高位吊颈线次日的涨停（诱多） |

### 2.2 涨停质量分 → 系统如何处理

```
质量分 >= 80: "龙头首板" — 最高优先级推荐，放宽PE/量价限制
质量分 60-80: "强势涨停" — 正常推荐，标准过滤
质量分 40-60: "跟风涨停" — 谨慎推荐，提高过滤门槛
质量分 < 40: "可疑涨停" — 不推荐（尾盘板、高位加速板）
```

### 2.3 涨停不可被简单毙掉的条件

当前"涨停→过热扣分→被毙"的链条需要打断：

```
旧逻辑: 量价-30 → fatal → SELL
新逻辑: 量价-30 且 涨停质量>=60 → 降为WARNING（不毙）
       量价-30 且 涨停质量<40  → 维持SELL
```

同样的涨停，质量不同结论不同：
- **诺德股份(+10%)**：要查封板时间+封单强度。如果是开盘秒板+大单封死 → 不能毙
- **盛剑科技(+10%)**：同样查。如果是尾盘偷袭+炸板多次 → 该毙

## 3. 板块龙头识别——涨停带动效应

### 3.1 谁是龙头？

同一板块内多只涨停，最先封板、封单最足的那只就是龙头。

```
PCB板块 6月17日涨停梯队:
  诺德股份  10:02封板 封单1.2亿  ← 最早封板 = 龙头
  光华科技  10:35封板 封单0.5亿  ← 跟风
  鹏鼎控股  13:15封板 封单0.8亿  ← 午后跟风
```

**龙头溢价**：板块龙头即使技术面有瑕疵，也应该比跟风票获得更高推荐优先级。

### 3.2 板块→涨停→联动→锚定

```
铜箔细分（PCB的子分支）:
  诺德股份(600110) 涨停 ← 铜箔龙头
  光华科技(002741) 涨停 ← 铜箔跟风
  铜冠铜箔(301217) +2%  ← 铜箔没涨停但被带动
  
分析逻辑:
  1. 诺德最早涨停 → 铜箔龙头
  2. 光华跟风涨停 → 铜箔确认
  3. 铜冠虽没涨停 → 但同细分，被龙头锚定，次日有补涨预期
```

## 4. 涨停数据采集

需要的涨停相关数据（当前系统全缺）：

| 数据 | 来源 | 用途 |
|------|------|------|
| 封板时间 | 腾讯财经/东财分时 | 判断涨停质量 |
| 封单量 | mootdx盘口 | 判断封单强度 |
| 炸板次数 | mootdx逐笔 | 判断封板稳定性 |
| 涨停原因 | 同花顺热点reason tags | 判断题材驱动 |
| 板块内涨停排序 | 同板块股票对比 | 判断龙头/跟风 |

**简化方案（不依赖逐笔数据）**：
- 封板时间：用当日分钟K线判断（第一根涨停价的分钟K出现时间）
- 封单强度：用 `limit_up` 价格 × 卖一价位的成交量估算
- 炸板：用分钟K线触及涨停价后又回落来判断
- 涨停原因：同花顺 `ths_hot_reason()`
- 板块排序：同板块内对比涨停时间

## 5. 鹏鼎控股的归属问题

**现状**：鹏鼎控股(002938) 在共振列没排上号，在瓶颈列（封装基板）出现。

**根因**：`pick_representative_stocks` 按涨幅+量比+多头打分排序，鹏鼎+10%涨停本该高分，但 PCB 板块有 174 只候选，涨停的不止它一只，它在共振列被其他涨停票挤掉了。

**修复**：
1. 共振列选股加入"辨识度"权重（涨停+市值+机构覆盖）
2. 鹏鼎控股同时满足 PCB（共振）和封装基板（瓶颈），应该双列显示
3. 跨列标的应标注"多轨共振"增强信号

## 7. 关键设计决策

1. **涨停不是风险信号，是需要分析的事件**。封板时间+封单强度+板块地位决定了它是"真机会"还是"真陷阱"
2. **龙头溢价真实存在**。最早涨停的票即使技术面有瑕疵，也应该比跟风票优先推荐
3. **涨停联动产生补涨机会**。同细分板块的涨停龙头 → 同细分未涨停标的次日有补涨预期
4. **数据源限制**。封板时间和封单强度需要分钟K线或盘口数据，如果数据不全则降级为"简化评分"
