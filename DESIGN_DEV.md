# 推荐系统增强开发文档 v1

## 1. 问题诊断

### 1.1 当前痛点

**症状**：推荐股票是今日强势股，但次日表现往往不佳（追高被套）。

**根因分析**：

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 因子体系偏动量/反转 | 🔴 高 | 动量+反转+板块动量占权重 ~30%，天然选出近期强势股，缺乏对"强势能否持续"的判断 |
| Agent 分析形同虚设 | 🔴 高 | `multi_agent_analyzer.py` 只做了简单数据采集+4维度打分，**从未真正调用** trading-agents-plugin 的 7-agent 辩论管线 |
| 缺少 K 线形态识别 | 🟡 中 | `indicator_engine.py` 只有均线排列/突破/粘合的简单布尔判断，没有经典 K 线组合识别（锤子线、吞没、三兵等） |
| 缺少量价关系分析 | 🟡 中 | 只有量比和异常量，没有量价配合/背离/放量滞涨/缩量止跌等判断 |
| 缺少筹码分布 | 🔴 高 | 完全没有筹码分析。A 股中筹码分布是判断支撑/压力/主力意图的核心工具，缺失导致无法判断当前位置的安全性 |
| 缺少次日预测能力 | 🟡 中 | 整个体系做的是"截面排序"（cross-sectional ranking），不是"时序预测"。排序告诉你谁好，不告诉你明天会不会涨 |

### 1.2 核心矛盾

```
当前逻辑：因子高分 → 推荐买入
真实情况：因子高分 ≠ 明天会涨

为什么？
- 板块动量 5d 高分 → 板块已经涨了 5 天 → 短期回调概率大
- 底部突破 高分 → 当天放量大涨 → 次日获利盘涌出
- 没有筹码分析 → 不知道突破位上方是否有密集套牢盘
- 没有量价分析 → 不知道今天的放量是主力吸筹还是出货
```

## 3. 改进线 A：集成 7-Agent 辩论管线

### 3.1 目标

将 trading-agents-plugin 的完整 7-agent 管线接入 `unified_pipeline.py`，对每只候选股做真正的多维度对抗分析，而非当前的简单数据采集。

### 3.2 当前 vs 改造后

```
当前 multi_agent_analyzer.py:
  ① fetch_technical() → 取K线数据
  ② fetch_fundamentals() → 取PE/PB
  ③ fetch_news() → 取新闻标题
  ④ fetch_a_share_macro() → 取北向/宽度
  ⑤ _build_composite() → 简单4维度加减分
  ❌ 没有 Agent 辩论
  ❌ 没有 Bull/Bear 对抗
  ❌ 没有 Risk 压力测试
  ❌ 没有 Trader 定进出场

改造后:
  Phase 1 (并行): Technical + News + Fundamentals + Macro 4个Agent
      各自抓取数据并写出结构化分析报告
  Phase 2 (串行): Bull → Bear(直接反驳Bull) → Risk(压力测试)
      三方对抗辩论
  Phase 3: Research Manager 综合三方意见 → 推荐
  Phase 4: Trader → 入场价/止损价/仓位
  Phase 5: Portfolio Manager → 最终 BUY/HOLD/SELL
  ✅ 完整 7-Agent 对抗分析
```

### 3.3 集成方案

```
unified_pipeline.py Step 3 改造:

当前:
  multi_agent_analyzer.analyze_stocks(codes) → 简单数据采集

改造后:
  对双轮选股输出的每只候选股:
    ① 用 fetch_a_share_data.py 准备4维度数据 (已有)
    ② 调用 trading-agents-plugin 的 Agent 子进程:
       - 对每只股票启动完整的 Phase 1-5
       - 将 Agent 输出解析为结构化 JSON
    ③ 汇总所有股票的 Agent 分析结果
    ④ 根据 PM 最终 SIGNAL 过滤:
       - BUY → 保留推荐
       - HOLD → 降级为观察
       - SELL → 从推荐列表剔除
```

### 3.4 具体改动

**文件**: `scripts/multi_agent_analyzer.py`
- 新增 `run_agent_pipeline(ticker, data)` 函数，调用 trading-agents 的 fetch 脚本 + 编排 Agent 调用
- 保留现有 `analyze_single_stock()` 作为数据准备层
- 新增 `AgentResult` 数据结构: `{ticker, signal, entry, stop, size, bull_case, bear_case, risk_level, verdict}`

**文件**: `scripts/unified_pipeline.py`
- Step 3 从 `analyze_stocks()` 改为 `run_full_agent_analysis()`
- 增加 `--agent-depth` 参数: `quick`(单Agent) / `full`(7-Agent完整辩论)
- Agent 结果注入最终排名（BUY +5分，HOLD +0，SELL -10分）

## 4. 改进线 B：新增深度技术分析模块

### 4.1 K 线形态识别 (`candlestick_patterns.py`)

在 `indicator_engine.py` 的基础上新增经典 K 线组合识别。

#### 4.1.1 反转形态（最重要）

| 形态 | 方向 | 信号强度 | 识别条件 |
|------|------|---------|---------|
| **锤子线** | 看多 | ⭐⭐⭐ | 实体小、下影≥2倍实体、在下跌趋势末端 |
| **倒锤子** | 看多 | ⭐⭐ | 实体小、上影≥2倍实体、在下跌趋势末端、次日阳线确认 |
| **看涨吞没** | 看多 | ⭐⭐⭐⭐ | 阴线后阳线、阳线实体完全包住阴线实体 |
| **启明星** | 看多 | ⭐⭐⭐⭐⭐ | 阴线→小实体星线(跳空低开)→阳线(跳空高开,收盘回补) |
| **穿刺线** | 看多 | ⭐⭐⭐ | 阴线后阳线低开、阳线收盘超过阴线实体的50% |
| **三白兵** | 看多 | ⭐⭐⭐⭐ | 连续三根阳线、每根收盘高于前一根、实体逐渐放大 |
| --- | --- | --- | --- |
| **吊颈线** | 看空 | ⭐⭐⭐ | 实体小、下影≥2倍实体、在上涨趋势末端 |
| **射击之星** | 看空 | ⭐⭐ | 实体小、上影≥2倍实体、在上涨趋势末端 |
| **看跌吞没** | 看空 | ⭐⭐⭐⭐ | 阳线后阴线、阴线实体完全包住阳线实体 |
| **黄昏星** | 看空 | ⭐⭐⭐⭐⭐ | 阳线→小实体星线(跳空高开)→阴线(跳空低开) |
| **乌云盖顶** | 看空 | ⭐⭐⭐ | 阳线后阴线高开、阴线收盘低于阳线实体的50% |
| **三乌鸦** | 看空 | ⭐⭐⭐⭐ | 连续三根阴线、每根收盘低于前一根 |

#### 4.1.2 持续形态

| 形态 | 方向 | 识别条件 |
|------|------|---------|
| 上升三法 | 看多 | 长阳→3根小阴(在阳线实体内)→长阳突破 |
| 下降三法 | 看空 | 长阴→3根小阳(在阴线实体内)→长阴跌破 |
| 平台突破 | 看多 | 横盘≥10日、振幅<8%、放量突破上沿 |

### 4.2 量价关系分析 (`volume_price_analyzer.py`)

#### 4.2.1 分析维度

```
维度1: 量价配合/背离
  - 价涨量增 → 健康上涨，+10分
  - 价涨量缩 → 量价背离，上涨乏力，-5分
  - 价跌量缩 → 正常回调，+5分
  - 价跌量增 → 放量下跌，危险，-15分

维度2: 关键量的识别
  - 地量（近60日最低量）→ 抛压枯竭，关注反转
  - 天量（近60日最高量）→ 分歧极大，可能是顶或底
  - 倍量（量比>2）→ 关注方向

维度3: 放量位置判断
  - 低位倍量阳线 → 主力吸筹，+15分
  - 高位倍量阳线 → 主力对倒诱多，-10分
  - 突破位倍量 → 有效突破确认，+10分
  - 跌破位倍量 → 恐慌出逃，-15分

维度4: 缩量的意义
  - 上涨后缩量回调 → 洗盘健康，+10分
  - 下跌后缩量横盘 → 筑底，+8分
  - 连续缩量阴跌 → 无人接盘，-10分

维度5: 量堆识别
  - 识别连续3天以上放量的"量堆"
  - 量堆位置（高位/低位/突破位）
  - 量堆后的走势（继续涨/开始跌）
```

### 4.3 筹码分布分析 (`chip_distribution.py`)

这是 A 股技术分析中最重要的缺失模块。

#### 4.3.1 什么是筹码分布

```
筹码分布 = 在不同价格区间的持仓成本分布

核心概念:
  - 筹码峰: 某个价格区间有大量成交 → 密集成本区
  - 筹码谷: 两个筹码峰之间的低成交区间 → 真空区（股价容易快速通过）
  - 获利盘比例: 当前价下方筹码占总筹码的百分比
  - 套牢盘比例: 当前价上方筹码占总筹码的百分比
  - 筹码集中度: 前10%股东的筹码占比
```

#### 4.3.2 分析维度

```
维度1: 筹码峰位置与股价关系
  - 股价在筹码峰上方 → 下方有支撑，+10分
  - 股价在筹码峰下方 → 上方有压力，-10分
  - 股价在筹码峰中间 → 方向不明

维度2: 获利盘比例
  - 获利盘 > 80% → 短期获利回吐压力大，-8分
  - 获利盘 30-70% → 正常博弈区间
  - 获利盘 < 20% → 深度套牢，反弹难度大，-15分

维度3: 筹码集中度变化
  - 筹码趋于集中（股东数下降）→ 主力吸筹，+15分
  - 筹码趋于分散（股东数上升）→ 主力出货，-15分
  - 数据源: 每季度股东户数变化

维度4: 套牢盘密集度
  - 上方有密集筹码峰 → 上涨阻力大（套牢盘解套抛压）
  - 上方筹码稀疏 → 突破后空间大
  - 距上方筹码峰越近 → 越容易遇阻

维度5: 底部筹码锁定
  - 底部筹码峰未松动 + 股价上涨 → 主力未出货，+20分
  - 底部筹码峰快速消失 + 股价横盘 → 主力出货，-20分
```

#### 4.3.3 近似计算方法

精确的筹码分布需要逐笔成交数据（交易所 L2 数据），我们使用近似方法：

## 5. 改进线 C：增加次日预测因子

### 5.1 过热惩罚因子

当前体系选出的是"最强"的股票，但往往也是"最热"的股票，短期回调概率大。

```python
# 在 factor_library.py 中新增

def _score_overheat_penalty(df, ind) -> float:
    """
    过热惩罚分：多个维度判断是否短期过热。

    触发条件（满足越多扣越多）：
      - 近5日涨幅 > 15%: -15分
      - 近5日涨幅 > 10%: -8分
      - 量比 > 5（异常放量）: -10分
      - 连续3日阳线且每日涨幅 > 3%: -10分
      - RSI > 80: -10分
      - J值 > 100（KDJ超买）: -8分
      - 换手率 > 25%: -12分
      - 偏离 MA20 > 15%: -10分

    最高扣分: -40分
    """
```

### 5.2 回调风险预测分

```python
def _score_pullback_risk(df, ind) -> float:
    """
    预测次日回调的风险。

    依据:
      - 当日是否在阻力位附近（MA60/MA120/筹码峰）
      - 当日上涨是否缩量（涨但是没人跟）
      - 上影线长度占比（冲高回落程度）
      - 尾盘是否走弱（最后30分钟 vs 全天走势）
      - 板块是否过热（板块涨幅排名前5%）

    返回: 0（无回调风险） ~ 100（极高回调风险）
    """
```

## 9. 关键设计决策

1. **Agent 分析放在最后一道**：只对 Top 10-15 做 Agent 分析（成本控制），避免对全市场数千只股票启动 Agent
2. **筹码分布用近似算法**：真正的筹码分布需要 L2 逐笔数据（年费数万），我们用 VWAP+成交量近似已经很接近实战需求
3. **K线形态独立成模块**：和 indicator_engine 的技术指标分开，形态是"定性"判断，指标是"定量"计算
4. **量价分析的核心是"位置判断"**：同样的放量，在底部和顶部的含义完全不同，必须结合价格位置做判断
5. **过热惩罚是纠偏机制**：当前体系偏向选出"已经涨了很多"的股票，过热惩罚因子主动降权这些标的

## 11. 风险与注意事项

- **Agent 调用成本**：7-Agent 管线每只股票约 60s 和大量 token 消耗，仅用于 Top 10-15
- **筹码分布精度**：近似算法无法替代真实 L2 数据，标注为"估算"而非"精确"
- **K线形态可靠性**：单一形态信号弱，需要多形态共振+量确认才有意义
- **过度优化风险**：新因子加入后需要通过回测验证，避免针对近期行情过度调参
- **数据依赖**：`chip_distribution.py` 需要至少 120 天 K 线数据才能有效估算
