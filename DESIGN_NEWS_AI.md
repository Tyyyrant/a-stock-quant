# 新闻AI推理引擎 — 开发文档

## 1. 当前瓶颈

`news_ripple.py` 是**死的关键词匹配**：

```python
PRODUCT_KEYWORDS = {"六氟化钨": "六氟化钨", "光刻胶": "光刻胶", ...}
```

问题：新闻必须包含这些精确关键词才会触发。95%的新闻（基金发行、暴雨、政策表态、进出口数据...）被直接丢弃。

## 2. AI推理方案

### 三层推理模型

```
新闻快讯（30条/天）
  │
  ├─ Layer 1: 快速分类（LLM 一次调用）
  │   "基金发行刷新纪录" → 类型=增量资金, 相关板块=证券/ETF重仓
  │   "暴雨" → 类型=自然灾害, 相关板块=水利/防汛/农业
  │   "FSD禁止" → 类型=政策风险, 相关板块=智能驾驶/特斯拉链
  │
  ├─ Layer 2: 深度推理（Agent 辩论）
  │   对每条高价值新闻，多Agent辩论：
  │   · 影响范围Agent：利好/利空哪些一级行业？
  │   · 供应链Agent：沿产业链向上/下游传导？
  │   · 时效Agent：短期脉冲还是中期趋势？
  │
  └─ Layer 3: 标的映射
      推理结果 → A股概念板块 → 成分股 → 技术面筛选
      → 注入 pipeline 作为 "新闻推理" 轨道
```

### 提示词示例

```
你是一个A股新闻分析师。分析以下新闻对A股的影响：

"单日37只！基金发行刷新历史纪录"

请回答：
1. 这些基金主要是什么类型？（股票/债券/混合/ETF）
2. 如果是股票型，增量资金可能流向哪些板块？
3. 利好哪些A股概念板块？（用申万一级行业或东财概念板块名称）
4. 影响是短期脉冲还是中期趋势？
5. 建议关注的具体标的（3-5只）
```

## 3. 集成方案

### 新增文件

| 文件 | 作用 |
|------|------|
| `scripts/news_ai_reasoning.py` | LLM驱动的新闻推理引擎 |

### 调用流程

```python
# quick_trade.py Layer 3.7 增强
news_items = eastmoney_global_news(50)
ai_reasoning = news_ai_reasoning.analyze(news_items)  
# → [{topic, sectors, stocks, confidence, reasoning}]

for r in ai_reasoning:
    if r['confidence'] > 0.6:
        # 注入候选池，标注 "新闻推理"
        all_picks.append({..., "sector": f"新闻推理:{r['topic']}"})
```

## 4. 实施计划

| 阶段 | 内容 | 耗时 |
|------|------|------|
| Phase 1 | 用 Claude Agent 做单次新闻分析验证 | 0.5h |
| Phase 2 | 编写 `news_ai_reasoning.py`，接入 LLM | 0.5天 |
| Phase 3 | 集成到 pipeline，新增"新闻推理"轨道 | 0.5天 |

## 5. 与现有新闻涟漪的关系

| 维度 | 新闻涟漪 (keyword) | 新闻AI推理 (LLM) |
|------|-------------------|-----------------|
| 触发方式 | 关键词匹配 | LLM理解语义 |
| 覆盖率 | ~5%新闻 | ~80%新闻 |
| 深度 | 只有直接替代 | 推理传导链 |
| 延迟 | 实时 | ~3-5秒/条 |
| 成本 | 免费 | LLM token费用 |
